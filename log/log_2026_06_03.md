suspend fun deleteAllWordsInSet(setId: String) {
    val user = account.get()
    var lastDocId: String? = null

    while (true) {
        val queries = mutableListOf(
            Query.equal("userId", user.id),
            Query.equal("setId", setId),
            Query.limit(500)
        )
        lastDocId?.let { queries.add(Query.cursorAfter(it)) }

        val result = databases.listDocuments(...)
        if (result.documents.isEmpty()) break

## 🟠 HIGH #4: Tag search dùng `Query.equal` trên array field

### Mô tả lỗi
**File:** `AppwriteVocabularySetRepository.kt:94` (cũ)

```kotlin

`Query.equal("tags", "Business")` trên array field kiểm tra **toàn bộ array == "Business"**, không phải **"Business" nằm trong array**. Tag `"Business"` sẽ không match document có `tags = ["Business", "TOEIC"]`.

### Cách sửa

} catch (error: Exception) {
    _uiState.update { it.copy(errorMessage = "...") }
}
```

`catch (error: Exception)` bắt tất cả Exception, bao gồm `CancellationException`. Khi user rời màn hình (coroutine bị cancel), thay vì âm thầm hủy, code set `errorMessage` — gây ra error flash không mong muốn.

### Cách sửa
Bắt riêng `CancellationException` và re-throw:

```kotlin
} catch (error: CancellationException) {
    throw error  // re-throw để coroutine framework xử lý cancel đúng cách
} catch (error: Exception) {
    _uiState.update { it.copy(errorMessage = "...") }
}
```

### Bài học
Trong Kotlin coroutine, `CancellationException` là tín hiệu bình thường (không phải lỗi). Luôn re-throw nó thay vì bắt và xử lý như lỗi. Pattern chuẩn: `catch (e: CancellationException) { throw e } catch (e: Exception) { /* xử lý lỗi thật */ }`.

private fun wordPermissions(ownerId: String, isPublic: Boolean): List<String> {
    val perms = mutableListOf(
        Permission.read(Role.user(ownerId)),
        Permission.update(Role.user(ownerId)),
        Permission.delete(Role.user(ownerId))
    )

}
